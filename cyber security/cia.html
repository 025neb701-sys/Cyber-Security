he CIA Triad is the cornerstone framework of information security, representing three fundamental goals that every secure system must achieve: Confidentiality, Integrity, and Availability. Confidentiality ensures that sensitive data is accessible only to those who are authorised to see it — preventing unauthorised disclosure whether through hacking, eavesdropping, or careless handling. Integrity guarantees that data remains accurate and unaltered except through approved processes, protecting against both malicious tampering and accidental corruption. These two pillars together form the basic promise of any trustworthy information system. Availability, the third pillar, ensures that systems and data are accessible to authorised users whenever they need them. A system is not truly secure if it is locked away so tightly that legitimate users cannot access it — availability demands resilience against attacks like Distributed Denial of Service (DDoS), hardware failures, and natural disasters. The CIA Triad is not merely academic; every security policy, technology, and control in the real world maps back to protecting one or more of these three properties. Understanding this triad allows individuals and organisations to assess their risks systematically and build defences that are balanced rather than skewed toward just one dimension. Importance The CIA Triad provides a universal language for security. Every breach — from a password leak to a website going offline — can be described and analysed using these three principles. It helps organisations prioritise what to protect and how. Confidentiality: A hospital stores patient medical records. Only doctors and nurses assigned to that patient can view them. Encrypting the records and using role-based access control enforces confidentiality. Integrity: A student submits an online exam. The system uses hash values to detect if any answers were altered after submission. If a hash mismatch is found, the submission is flagged. Availability: A bank's net banking portal must be available 24/7. Load balancers and backup servers ensure that even if one server fails, customers can still access their accounts.
